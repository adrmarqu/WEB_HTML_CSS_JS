Introducci

Git s un programari de control de versions distribut i descentralitzat que permet a un equip de desenvolupadors treballar sobre el mateix codi.

S'anomena distribut perqu cada membre de l'equip t una cpia completa del codi.

Els membres de l'equip poden enviar-se codi, rebre'l i desenvolupar funcionalitats de manera conjunta i separada del servidor central.

Algunes ventajas de fer-lo servir:

  • s l'estndard actual.
  • Codi collaboratiu, versionat i distribut.
  • Recuperaci de fitxers.
  • Major control.
  • Shorcuts i plugins .
  • Millora la productivitat.

Installaci:

  • Git.

Installaci a Windows (usant VSCode)

  • Descarrega l'installador de Git per a Windows des de: https://git-scm.com/download/win i executa'l.

  • Durant la installaci, accepta els parmetres per defecte o escull opcions recomanades (afegir Git al PATH, gestionar saltos de lnia amb core.autocrlf, etc.).

  • Un cop installat, obre Visual Studio Code i obre el terminal integrat (View Terminal o Ctrl+ )..

  • Verifica la installaci amb:

    git --version

  • Si el terminal de VSCode no troba git, indica el cam de l'executable a la configuraci de VSCode. Obre la paleta de comandes (Cmd/Ctrl+Shift+P) i escull Preferences: Open Settings (JSON) i afegeix (exemple):

    "git.path": "C:\Program Files\Git\bin\git.exe"

  • Opcional: installa extensions tils com GitLens per millorar l'experincia Git dins de VSCode.

Installaci a macOS (usant VSCode)

  • Opci 1 (recomanada si tens Homebrew): obre el terminal integrat de VSCode i executa:

    brew install git

  • Opci 2: installa les Xcode Command Line Tools executant en el terminal:

    xcode-select --install

  • Verifica la installaci des de VSCode amb:

    git --version

  • Si cal apuntar VSCode a un Git concret (per exemple Homebrew en Mac M1/M2), obre Preferences: Open Settings (JSON) i afegeix l'exemple:

    "git.path": "/opt/homebrew/bin/git"

Configuraci inicial

Configurant Git per primera vegada

git --version
git config --global user.name "Anscari Torrecilla"
git config --global user.email anscaritorrecilla@gmail.com
git config --global user.ui true
git config --global init.defaultBranch main
git config --list
# assignant vscode com a editor de configuraci de git
git config --global core.editor "code --wait"
git config --global -e
# para estandarizar los saltos de lnea en windows
git config --global core.autocrlf true
# para estandarizar los saltos de lnea en linux/mac
git config --global core.autocrlf input
# ver todas las opciones de la configuracin en la terminal
git config -h
# ver todas las opciones de la configuracin en el navegador
git help config

Configuraci des de Visual Studio Code

  1. Obre Visual Studio Code i navega a la carpeta del teu projecte (File Open Folder).

  2. Obre el terminal integrat (View Terminal o Ctrl+ ). i comprova que git est disponible:

    git --version

  3. Executa les ordres de configuraci globals des del terminal integrat per establir nom i correu, i altres opcions (exemple):

    git config --global user.name "Nom Cognoms" git config --global user.email correu@exemple.com git config --global core.editor "code --wait"

    Windows

    git config --global core.autocrlf true

    mac/linux

    git config --global core.autocrlf input git config --global init.defaultBranch main

  4. Ajustar la ruta de git a VSCode (si cal): obre la paleta (Cmd/Ctrl+Shift+P) Preferences: Open Settings (JSON) i afegeix per exemple:

    "git.path": "C:\Program Files\Git\bin\git.exe"

    o en macOS:

    "git.path": "/opt/homebrew/bin/git"

  5. Configuracions des de la interfcie de VSCode:

  • Pots canviar opcions relacionades amb Git des de Settings (Cmd/Ctrl+,) buscant Git. Per exemple, habilitar Git: Auto Fetch o ajustar Git: Confirm Sync.
  • Installa extensions com GitLens per visualitzar historial, blame, i millorar la integraci amb VSCode.

  1. Comprova la configuraci actual des del terminal integrat:

    git config --list

Amb aquests passos tindrs Git installat i configurat utilitzant l'entorn i les eines de Visual Studio Code.

Flux bsic

El flux de Git consta de tres estats locals, s a dir a l'ordinador on s'est treballant i un ms de forma remota quan accedim al codi centralitzat en plataformes com GitHub , Gitlab , Bitbucket , etc.

Aquests estats sn modified , staged , committed i remote .

A cadascuna li correspon una rea de treball:

  1. Working Directory : s l'rea corresponent a l'estat modified i s la carpeta local del teu ordinador on emmagatzemis els fitxers del teu projecte.
  2. Staging Area : s l'rea corresponent a l'estat staged tamb s'anomena index perqu s l'rea on git indexa i afegeix els canvis realitzats als fitxers previs a comprometre'ls al seu registre.
  3. Local Repository : s l'rea corresponent a l'estat committed , on els canvis ja s'han registrat al repositori de git tamb s'anomena HEAD perqu indica en quin canvi es troba el punter del repositori.
  4. Remote Repository : s l'rea corresponent a l'estat remot i s el directori remot on emmagatzemem els fitxers del projecte en alguna plataforma web com GitHub , GitLab , BitBucket . Git anomena origin el repositori remot.

#agregar els canvis d'un arxiu al staged
git add arxiu/directori

#agregar tots els canvis de tots els arxius al staged
git add .

#els canvis sn compromesos en el repositori.
#has d'escriure el missatge del canvi quan s'obri l'arxiu de configuraci, en acabar guarda i tanca l'arxiu perqu tinguin efecte.
git commit

#s un shortcut del comando anterior, escrius i confirmes el missatge del canvi en un pas
git commit -m "missatge descriptiu del canvi"

#s'agrega l'origen remot del teu repositori de GitHub
git remote add origin https://github.com/usuari/repositori.git

#la primera vegada que vinculem el repositori remot amb el local
git push -u origin main

#per a les subseqents actualitzacions, si n canvies de branca
git push

#per a descarregar els canvis del repositori remot al local
git pull

De mster a main

Amb els desafortunats esdeveniments del 25 de maig de 2020 als Estats Units que van culminar amb l'assassinat de l'afroameric George Floyd a mans de policies de la ciutat de Minepolis , es va intensificar de manera global el moviment #BlackLivesMatter .

Amb aquest moviment moltes indstries i empreses van comenar a prendre accions per erradicar el racisme.

A la indstria de la tecnologia per anys s'han emprat paraules com master , slave , whitelist , blacklist entre d'altres que actualment no sn ben vistes pel context i la semntica que impliquin.

Sobre aix Microsoft empresa propietaria de GitHub va decidir comenar una campanya per reemplaar el nom de la branca principal dels repositoris de mster a main ; com ho han explicat en aquest document :

"L'1 d'octubre de 2020, qualsevol nou repositori que cres utilitzar ' main ' com la rama per defecte, en lloc de ' master '. Aquest canvi no afecta cap dels teus repositoris existents: els repositoris existents continuaran tenint la mateixa rama per defecte que tenen ara.

Aquest canvi implica afegir un parell de lnies d'ordres addicionals per crear la branca ' main ' i fer-la principal al repositori. El flux bsic quedar de la segent manera:

Per a repositoris nous

git init
git add .
git commit -m "Primer commit"
git branch -M main
git remote add origin https://github.com/usuari/repositori.git
git push -u origin main

Per a repositoris existents

git branch -M main
git remote add origin https://github.com/usuari/repositori.git
git push -u origin main

Per reemplaar la branca master per main a GitHub

#Pas 1
#Crea la branca local main i passa-li l'historial de la branca mster
git branch -m master main

#Pas 2
#Fes un push de la nova branca local main    en el repositori remot de GitHub
git push -u origin main

#Pas 3
#Canvia el HEAD actual a la branca main
git symbolic-ref refs/remotes/origin/HEAD refs/remotes/origin/main

#Pas 4
#Canvia la branca_default_ de _mster_ a _main_ en el teu repositori de _GitHub_ .

Per a fer-ho, segueix les instruccions d'aquest enlla.

# Pas 5
#Elimina la branca mster del repositori remot
git push origin --delete master

Per a reemplaar la rama master per main en Git

git config --global init.defaultBranch main

Branques

Una branca ens permet allar una nova funcionalitat en el nostre codi que desprs podrem afegir a la versi principal.

#crear branca
git branch nom-branca

#canviar de branca
git checkout nom-branca

#crear una branca i canviar-te a ella
git checkout -b branca

#eliminar branca
git branch -d nom-branca

#eliminar branques remotes
git push origin --delete nom-branca

#eliminar branca (forat)
git branch -D nom-branca

#llistar totes les branques del repositori
git branch

#llistar les branques no fusionades a la branca actual
git branch --no-merged

#llesta branques fusionades a la branca actual
git branch --merged

#depassar branques
git checkout branca-secundria
git depassi branca-principal

Fusions

Uneix dues branques. Per a fer una fusi necessitem:

  1. Situar-nos en la branca que es quedar amb el contingut fusionat.
  2. Fusionar. Quan es fusionen branques es poden donar 2 resultats diferents:

  • *_*Fast-*Forward_**: La fusi es fa automtica, no hi ha conflictes per resoldre.

  • *Manual Merge**: La fusi cal fer-la manual, per a resoldre conflictes de duplicaci de contingut.

    #ens canviem a la branca principal que quedar de la fusi git checkout branca-principal

    #executem el comando merge amb la branca secundria a fusionar git merge branca-secundria

Registre de l'historial

*git *log ens permet conixer tot l'historial d'un projecte, amb la informaci de la data, l'autor i aneu de cada canvi.

git log

#mostra en una sola lnia per canvi
git log --oneline

#guarda el log en la ruta i arxiu que especifiquem
git log > commits.txt

#mostra l'historial amb el format que indiquem
git log --pretty=format:"%h - %an, %ar : %s"

#canviem la n per qualsevol nombre enter i mostrar els n canvis recents
git log -n

#mostra els canvis realitzats desprs de la data especificada
git log --after="2019-07-07 00.00:00"

#mostra els canvis realitzats abans de la data especificada
git log --before="2019-07-08 00.00:00"

#mostra els canvis realitzats en el rang de data especificat
git log --after="2019-07-07 00.00:00" --before="2019-07-08 00.00:00"

#mostra una grfica de l'historial de canvis, branca i fusions
git log --oneline --graph --all

#mostra tot el registre d'accions del log
#incloent insercions, canvis, eliminacions, fusions, etc.
git reflog

#diferncies entre el Working Directory i el Staging Area
git diff

Colaboraci a GitHub

Per poder collaborar en projectes allotjats en GitHub necessitem fer s dels forks i pull requests, eines que ens ofereix la plataforma per a aquest objectiu.

A continuaci descric el procs de collaboraci en GitHub.

  1. Forkea el repositori en el qual vulguis collaborar, per a fer-lo, segueix les instruccions d'aquest enlla.
  2. Una vegada forkeat el repositori en el teu compte de GitHub, clona'l en el teu equip de cmput.
  3. En el repositori local cal configurar els orgens remots de la teva nova copa per a tenir tots dos remots, els originals (origin) i els de la teva cpia, per a fer-ho, segueix les instruccions d'aquest enlla.
  4. Crea una branca nova en la teva fork local per a fer la teva collaboraci, i sincrnizala amb el teu repositori remot, per a fer-ho, segueix les instruccions d'aquest enlla.
  5. Configura el teu repositori perqu vaig acceptar canvis (pull requests), per a fer-ho, segueix les instruccions d'aquest enlla.
  6. Crea una pull request, per a fer-ho, segueix les instruccions d'aquest enlla.
  7. Espera al fet que l'amo del repositori original, accepti els teus canvis.
  8. Una vegada que acceptin la teva pull request, s recomanable que esborris la branca en la qual vas treballar el canvi i actualitzis el teu repositori forkeado, amb els canvis del repositori original.

Resum dels comandos a executar per a collaborar en un repositori de GitHub:

# forkear repositori
git clone https://github.com/usuario/repositorio.git
git remote -v
git remote rename origin fork
git remote add origin https://github.com/usuario/repositorio.git
git checkout -b branca-nova
git push fork branca-nova

# sollicitar el pull request
# acceptar el pull request
git checkout main
git pull origin main
git push fork main
git branch -d branca-nova
git push fork --delete branca-nova
